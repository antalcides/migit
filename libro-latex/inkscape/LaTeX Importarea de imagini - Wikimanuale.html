Pachetul graphicx[modificare]

Dup cum am spus mai devreme, LaTeX nu poate gestiona imaginile direct, aadar vom folosi faciliti adiionale: trebuie s ncrcm pachetul graphicx n preambulul documentului:

\usepackage{graphicx}

Acest pachet accept ca argument driverul extern de utilizat pentru gestionarea imaginilor; totui, ultima versiune a acestui pachet se ocup de toate aspectele, schimbnd driverul n funcie de compilatorul folosit, astfel c nu trebuie s v ocupai personal de aceste lucruri. i totui, presupunnd c vrei s nelegei mai bine cum merge, iat opiunile posibile pe care le putei pasa pachetului:

  • dvips (implicit dac compilai cu comanda latex), dac facei compilarea cu latex pentru a obine un DVI i vrei s vizualizai documentul cu un program pentru DVI sau PS.
  • dvipdfm, dac compilai cu latex pentru a obine un DVI, pe care intenionai apoi s-l convertii ntr-un PDF cu comanda dvipdfm, putnd vizualiza documentul cu orice program pentru PDF.
  • pdftex (implicit dac realizai compilarea cu pdflatex), dac compilai cu pdftex pentru a obine un PDF pe care l putei vedea cu un program de vizualizare pentru PDF-uri.

Din nou, nu trebuie s pasai nicio opiune pachetului, ntruct setrile implicite sunt destul de bune n cele mai multe cazuri.

n multe privine, importarea imaginilor n document folosind sistemul LaTeX este destul de simpl, ndat ce avei imaginile n formatul corespunztor! De aceea, m tem c pentru cei mai muli utilizatori, efortul cel mai mare va fi convertirea fiierelor grafice. n continuare vom vedea ce formate pot avea imaginile pe care le putei include, iar apoi vom vedea cum putei face asta.

Opiunile documentului[modificare]

Pachetele graphics i graphicx recunosc opiunile "draft" i "final" date n comanda \documentclass[...]{...} la nceputul fiierului. Opiunea "draft" va mpiedica includerea imaginii n fiierul de ieire i va nlocui coninutul cu numele fiierului ce conine imaginea. Opiunea "final" va face ca imaginea s fie inclus n fiierul de ieire. Valoarea implicit este "draft".

Formate suportate de imagini[modificare]

Dup cum am explicat mai devreme, formatele de imagini pe care le vom putea folosi depind de driverul utilizat de graphicx, ns, de vreme ce driverul este ales n mod automat n funcie de compilator, formatele permise de imagini vor depinde de compilatorul folosit.

Compilarea cu latex[modificare]

Singurul format pe care l putei include cnd compilai cu comanda latex este Encapsulated PostScript (EPS).

Formatul EPS a fost definit de firma Adobe Systems pentru a facilita importarea de ctre aplicaii a elementelor grafice bazate pe formatul PostScript n documente. Deoarece un fiier EPS declar dimensiunea imaginii, faciliteaz sistemelor de genul LaTeX aranjarea textului i graficii. EPS este un format vectorial asta nseamn c poate avea o calitate foarte mare dac este creat n mod corespunztor cu programe de grafic vectorial. Putem stoca de asemenea imagini bit-map n format EPS, ns vor necesita foarte mult spaiu pe disc.

Multe pachete cu programe grafice au abilitatea de a salva imagini n format EPS (extensia e n mod normal .eps). Iat cteva exemple de programe ce pot genera fiiere EPS:

  • Tiprirea sub forma unui fiier EPS:
    • n Windows, PDFCreator este un program open source ce poate crea att fiiere PDF, ct i EPS. Instaleaz o imprimant virtual care poate fi accesat din alte programe ce au o intrare "print..." n meniu (practic orice program).
  • Crearea i convertirea elementelor de grafic vectorial:
    • Programele comerciale de grafic vectorial, de genul Adobe Illustrator, CorelDRAW i FreeHand pot citi i scrie fiiere EPS. Totui, aceste produse sunt limitate la platformele Windows i Mac OS.
    • Inkscape poate salva fiiere n formatul vectorial EPS i ruleaz pe platforme multiple. Inkscape nu poate deschide fiiere EPS direct; totui, cu programul utilitar epstopdf putei converti EPS n PDF, iar Inkscape poate importa PDF-uri. ncepnd cu versiunea 0.48, Inkscape are o opiune special de output PDF+LaTeX (i EPS/PS). Vezi site-ul Inkscape.
    • Dia este un program utilitar cross platform pentru diagrame, ce poate exporta EPS-uri.
  • Crearea i convertirea elementelor grafice bazate doar pe raster n EPS:
    • GIMP are o interfa grafic i este multi-platform.
    • Pentru linia de comand:
      • Sam2p (convert) sau
      • ImageMagick (convert) sau
      • GraphicsMagick (gm convert).
    • Aceste trei programe opereaz cam la fel, i pot face conversii ntre cele mai multe formate grafice. Sam2p este totui cel mai recent dintre ele i pare s ofere cea mai bun calitate, dar i s dea cele mai mici fiiere de ieire.
      • imgtops. Un mic program utilitar pentru grafic.
  • Crearea de reprezentri grafice vectoriale de calitate pentru publicaii:
    • Gnuplot, ce produce grafic tiinific din 1986.
    • R, figuri statistice i tiinifice.
    • Generic Mapping Tools (GMT), care genereaz hri i o serie de reprezentri grafice foarte personalizabile.
    • Gnumeric, ce genereaz foi de calcul, poate exporta SVG, EPS, PDF.
    • matplotlib, bibliotec pentru reprezentri grafice, scris n Python, ce poate exporta n formate PDF i EPS.

Trebuie s apelai la cteva trucuri pentru a putea importa alte formate dect EPS n documentele DVI, ns acestea sunt foarte complicate. Pe de alt parte, convertirea oricrei imagini n format EPS este foarte simpl, astfel c nu are rost s le lum n considerare aici.

Compilarea cu pdflatex[modificare]

Dac compilai cu pdflatex pentru a produce un PDF, avei mai multe opiuni. Nu putei insera fiiere EPS. Putei insera:

  • JPG, folosit foarte mult pe Internet, camere digitale, etc. Reprezint cea are bun alegere dac vrei s inserai fotografii.
  • PNG, un format foarte comun (chiar dac nu n aa msur ca JPG); este un format fr pierderi la dezarhivare (lossless) i reprezint cea mai bun alegere pentru diagrame (dac nu ai reuit s generai o versiune vectorial) i instantanee ale ecranului ('screenshots').
  • PDF, foarte mult folosit pentru documente, ns poate fi utilizat i pentru a stoca imagini. Suport att imagini vectoriale, ct i imagini bit-map, ns nu se recomand pentru ultimele, de vreme ce JPG sau PNG ofer acelai rezultat folosind mai puin spaiu pe disc.
  • Formate vectoriale, ce pot fi folosite cu Inkscape. Exist instruciuni legate de cum putei salva imaginile vectoriale n format PDF neles de LaTeX i putei face sistemul LaTeX s gestioneze automat stilurile i dimensiunile textului din imagini.

JPG i PNG sunt suportate de orice program de procesare de imagini, astfel c putei s folosii formatul preferat. Dac vrei s creai PDF-uri vectoriale de calitate nalt pe care s le ncorporai n documentul final PDF, putei utiliza programul Inkscape: acesta ofer suport pentru numeroase formate vectoriale, astfel c poate face conversii dintr-un format n altul. De asemenea, putei crea imaginile direct cu Inkscape. Dac vrei s realizai reprezentri grafice matematice, Gnuplot poate salva datele n orice format.

Notai c fiierele EPS nu pot fi folosite cu comanda pdflatex, totui pot fi convertite n format PDF cu ajutorul programului utilitar epstopdf, inclus n majoritatea distribuiilor LaTeX. Acesta poate fi chemat automat de LaTeX folosind pachetul epstopdf. n Windows, putei converti mai multe fiiere scriind urmtoarea linie ntr-un fiier batch (un fiier text cu extensia .BAT) situat n acelai director cu imaginile:

for %%f in (*.eps) do epstopdf %%f

i care poate fi rulat din linia de comand. Dac epstopdf produce pagini ntregi, n care sunt incluse undeva imaginile, utilizai

$ epstopdf --gsopt=-dEPSCrop foo.eps

sau ncercai programul utilitar ps2pdf

$ ps2pdf -dEPSCrop foo.eps

pentru a obine documentul final PDF.

Imaginile pot fi salvate n diverse formate n scopuri diferite. Spre exemplu, un director poate conine un fiier "diagram.pdf" pentru tiprirea la rezoluie nalt, n timp ce "diagram.png" poate fi folosit pentru previzualizarea pe monitor. Putei specifica imaginea de utilizat de programul pdflatex prin intermediul urmtoarei comenzi plasate n preambulul documentului:

\DeclareGraphicsExtensions{.pdf,.png,.jpg}

care specific fiierele ce vor fi incluse n document, dac exist fiiere cu acelai nume de baz, ns cu extensii diferite.

Includerea imaginilor[modificare]

Acum c am vzut ce formate putem include i cum putem gestiona aceste formate, este timpul s nvm cum le putem include n document. Dup ce ai ncrcat pachetul graphicx n preambul, putei include imagini cu comanda \includegraphics, a crei sintax se prezint dup cum urmeaz:

\includegraphics[atr1=val1, atr2=val2, ..., atr_n=val_n]{nume_imagine}

Dup cum sper c tii deja, argumentele din paranteze drepte sunt opionale, n vreme ce argumentele din acolade sunt obligatorii. Argumentul din acolade este numele imaginii. Scriei fiierul cu imaginea fr extensie. n felul acesta, compilatorul LaTeX va cuta formatele suportate de imagini din acel director i-l va alege pe cel mai bun (EPS dac outputul este DVI; JPEG, PNG sau PDF dac outputul este PDF). Varietatea atributelor posibile ce pot fi setate este destul de mare, astfel c le descriem n continuare numai pe cele mai cunoscute:

width=xx Specific limea dorit a imaginii importate la xx. Not: raportul dintre lime i nlime se va menine numai dac specificai doar una dintre dimensiuni.
height=xx Specific nlimea dorit a imaginii importate la xx.
keepaspectratio Acesta poate fi setat la valorile adevrat (true) sau fals (false). La valoarea adevrat, va redimensiona imaginea dup lime i nlime, ns nu va distorsiona imaginea, astfel c nu se vor depi nici limea, nici nlimea.
scale=xx Redimensioneaz imaginea cu factorul de scar dorit. De exemplu, 0,5 va reduce la jumtate dimensiunile imaginii, n vreme ce o valoare de 2 le va dubla.
angle=xx Aceast opiune poate roti imaginea cu xx grade (n sens trigonometric, invers acelor de ceasornic).
trim=l b r t Aceast opiune va decupa imaginea importat dup l la stnga (left), b din partea inferioar (bottom), r de la dreapta (right) i t din partea superioar (top), unde l, b, r i t sunt lungimi.
clip Pentru ca opiunea trim s mearg, trebuie s setai clip=true.
page=x Dac fiierul ce conine imaginea este un PDF cu mai multe pagini, acest parametru v permite s utilizai o alt pagin dect prima.

Pentru a folosi mai mult dect o opiune la un moment dat, separai opiunile prin virgule. Ordinea opiunilor conteaz. Spre exemplu, mai nti ar trebui s rotii imaginea (cu un unghi), iar apoi s-i specificai limea.

Imaginile incluse vor fi inserate doar acolo unde scriei codul, iar compilatorul le va trata ca pe nite "csue mari". Dup cum vom vedea ntr-un capitol ulterior, asta va genera un output de slab calitate, astfel c ar fi mai bine s plasai imaginile n obiectele flotante.

Exemple[modificare]

OK, a venit timpul s vedei graphicx n aciune. Iat cteva exemple:

\includegraphics{chick}

Aceast comand import pur i simplu imaginea, fr alt procesare suplimentar. Totui, este foarte mare (aadar n-o voi afia aici!). Deci, s-i reducem dimensiunile:

\includegraphics[scale=0.5]{chick}

Dimensiunile imaginii au fost reduse la jumtate. Dac vrei s fii mai specifici i s dai lungimi efective dimensiunilor imaginii, putei proceda n felul urmtor:

\includegraphics[width=2.5cm]{chick}

Putei specifica scara i n raport cu limea liniei din mediul local (\linewidth), cu limea textului de pe o pagin (\textwidth) sau cu nlimea textului de pe pagin (\textheight) (iamginile nu sunt afiate):

\includegraphics[width=0.5\linewidth]{chick}
 
\includegraphics[width=0.75\textwidth]{chick}
 
\includegraphics[height=0.75\textheight]{chick}

Pentru a face o rotaie (am redus de asemenea dimensiunile imaginii):

\includegraphics[scale=0.5, angle=180]{chick}

i n sfrit, iat un exemplu de decupare a imaginii dac vrei s evideniai o anumit zon a acesteia:

% ordinea parametrilor opiunii de decupare (''trim''): stnga jos dreapta sus
\includegraphics[trim = 10mm 80mm 20mm 5mm, clip, width=3cm]{chick}

Observai prezena opiunii clip, fr de care nu se poate efectua operaia de decupare.

Dup cum probabil ai observat, denumirea fiierului ce conine imaginea este dat ntotdeauna fr extensie: LaTeX va avea grij s obin versiunea corespunztoare din directorul curent. Considerai situaia urmtoare: ai adugat cteva imagini n format JPG documentului i l-ai compilat cu succes pentru a obine un format de ieire PDF. Acum l compilai pentru a obine un DVI, rulai comanda latex i obinei o mulime de erori... ntruct ai uitat s includei n director versiunile EPS ale imaginilor pe care vrei s le inserai. La nceputul acestei cri, am spus c aceeai surs LaTeX poate fi compilat att n format DVI, ct i PDF fr nicio modificare. Acest lucru este adevrat, atta vreme ct nu utilizai anumite pachete, iar graphicx este unul dintre acestea. n orice caz, nc putei folosi ambele compilatoare i cu documente ce includ imagini, atta timp ct v amintii s furnizai imaginile n ambele formate (EPS i unul dintre JPG, PNG i PDF).

Margini[modificare]

Putei crea margini n jurul unei imagini n LaTeX cu ajutorul comenzii fbox:

\setlength\fboxsep{0pt}
\setlength\fboxrule{0.5pt}
\fbox{\includegraphics{chick}}

Putei controla spaiul liber dintre margini i imagine cu comanda \setlength\fboxsep{0pt}, n cazul de fa valoarea 0pt nseamn lipsa unui asemenea spaiu liber, astfel nct marginile vor fi plasate strns n jurul imaginii. Putei controla grosimea marginilor cu comanda \setlength\fboxrule{0.5pt}.

Stocarea imaginilor[modificare]

Exist o cale s transmitei sistemului LaTeX unde s caute dup imagini: spre exemplu, poate fi util s stocai imaginile ntr-un singur loc pentru a putea fi utilizate n mai multe documente. Rspunsul l constituie comanda \graphicspath, care are un argument ce d calea unui director adiional n care vrei ca sistemul LaTeX s caute imagini cnd un fiier folosete comanda \includegraphics. Iat cteva exemple (/ de la sfrit este necesar):

\graphicspath{{c:\imagini~1\camera}}
\graphicspath{{c:/imagini~1/camera/}} *
\graphicspath{{/var/lib/imagini/}}
\graphicspath{{./imagini/}}
\graphicspath{{dir_imagini/}{alt_dir/}{al_treilea_folder/}}
\graphicspath{{imagini//}}

* merge n Windows XP.

V rugm consultai http://www.ctan.org/tex-archive/macros/latex/required/graphics/grfguide.pdf. Ultima comand caut recursiv fiierele datorit semnului dublu slash "//". n exemplul prezentat, avei un director numit "imagini" n acelai director cu fiierul .tex principal (adres relativ). n maniera aceasta, putei crea un arbore de directoare n cadrul acestuia ce ar putea de exemplu s faciliteze sortarea/gsirea/editarea unui numr mare de fiiere ce conin imagini. Reinei c aceast abordare poate acapara foarte multe resurse.

Dup cum ai observat, n primul exemplu am utilizat o form "sigur" (MS-DOS) a unui folder de imagini din Windows, ntruct, dup cum tii, e o idee proast s folosii nume de directoare ce conin spaii. Utiliznd ci absolute, \graphicspath face fiierele mai puin portabile, n timp ce dac folosii ci relative (ca n ultimul exemplu), nu ar trebui s avei vreo problem de portabilitate, ns reinei s nu utilizai spaii n denumirile de fiiere. Ca o alternativ, dac utilizai comanda pdflatex pentru compilare, putei folosi pachetul grffile, care v va permite s utilizai spaii n denumirile fiierelor.

Imagini ca figuri[modificare]

Exist multe scenarii n care ai putea dori s nsoii o imagine cu un titlu sau o legend i poate cu nite referine. Acest lucru se poate face cu ajutorul mediului figure. Codul urmtor prezint minimul necesar pentru a utiliza o imagine ca pe o figur.

\begin{figure}[htb]
\includegraphics{imagine.png}
\end{figure}

Codul de mai sus este relativ simplu i nu ofer mult funcionalitate. Codul ce urmeaz prezint o utilizare mai extins a mediului figure, care este aproape universal utilizat, oferind un titlu sau o legend i o etichet, centrnd imaginea i reducnd dimensiunile la 80% din limea textului.

\begin{figure}[htb]
\begin{center}
\leavevmode
\includegraphics[width=0.8\textwidth]{imagine.png}
\end{center}
\caption{O imagine incredibil}
\label{fig:imagine_incredibil}
\end{figure}

Mediul figure nu se folosete exclusiv pentru imagini. Putei gsi mai multe informaii despre acest mediu i utilizarea sa n capitolul Elemente flotante, figuri i note.

Text n jurul imaginilor[modificare]

Text poate de asemenea s fie pus n jurul imaginilor. (Acest lucru este deosebit de util dac includei imagini nalte.)

% seciune de import
\usepackage{wrapfig}
 
% seciune de coninut
\begin{wrapfigure}{r}{8cm} % "l" pentru stnga (''left'') sau
% "r" pentru dreapta (''right'') paginii, 8cm este limea figurii
\centering
\includegraphics[height=80mm]{imagine.jpg}
\caption{Titlul imaginii}
\label{img}
\end{wrapfigure}

Crearea de elemente grafice vectoriale[modificare]

TikZ/PGF[modificare]

Mai multe informaii introductive despre TikZ putei gsi n capitolul Crearea de imagini.

Putei realiza direct imagini cu comenzi TeX, folosind pachetul tikz: http://ftp.dante.de/tex-archive/help/Catalogue/entries/pgf.html Acesta include foarte mult documentaie i numeroase exemple.

  \centering
  \begin{tikzpicture}
    \draw[thick,rounded corners=8pt] 
(0,0) -- (0,2) -- (1,3.25) -- (2,2) -- (2,0) -- (0,2) -- (2,2) -- (0,0) -- (2,0);
  \end{tikzpicture}
  \caption{Titlul figurii}
  \label{fig:test}
\end{figure}

Putei gsi o colecie ampl de exemple la adresa: http://www.texample.net/tikz/

Putei gsi pachete bazate pe TikZ (de exemplu pentru trasarea circuitelor electrice) la aceast adres: http://ftp.dante.de/tex-archive/help/Catalogue/bytopic.html#pgftikzsection

Xfig[modificare]

Putei realiza grafic vectorial cu ajutorul programului Xfig (vezi capitolul Instalare), care poate fi exportat n formate potrivite pentru LaTeX. n Xfig, ndat ce ai salvat imaginea sub forma unui fiier test.fig, trebuie s-l exportai cu ajutorul meniului File > Export din fereastra principal a Xfig, iar apoi s selectai "Combined PS/Latex (both parts)" din lista de limbaje. Dac nu schimbai alte setri, n acelai director cu test.fig vor fi create dou alte fiiere, i anume: test.pstex_t i test.pstex. Putei plasa atunci figura ntr-un document LaTeX:

\begin{figure}
  \centering
  \input{./xfig/test.pstex_t}
  \caption{Titlul figurii}
  \label{fig:test}
\end{figure}

ipe[modificare]

Editorul extensibil de imagini Ipe este un program gratuit pentru grafic vectorial ce creeaz figuri n formate PDF i EPS. Spre deosebire de Xfig, Ipe reprezint fonturile LaTeX la dimensiunea lor corect pe ecran, ceea ce faciliteaz plasarea de etichete textuale la locul potrivit. Ipe are de asemenea numeroase moduri de apucare (snapping) spre exemplu, apucare de puncte, linii sau intersecii), folosite pentru construcii geometrice.

Inkscape[modificare]

nc un program pentru crearea de grafic vectorial, disponibil la adresa http://www.inkscape.org/. Merge cu fiiere Scalable Vector Graphics (SVG), dei poate exporta n multe alte formate, ce pot fi incluse n fiiere LaTeX, de genul EPS i PDF. ncepnd cu versiunea 0.48, exist opiunea de output combinat PDF/EPS/PS+LaTeX, ca n cazul programului XFig.

Editarea imaginilor EPS[modificare]

Dup cum a fost descris mai sus, putei importa coninut grafic n LaTeX din programe externe sub form de fiiere EPS. ns cteodat vei dori s editai sau s retuai aceste fiiere grafice. Un fiier EPS poate fi editat cu orice editor de texte, de vreme ce este scris n format ASCII. ntr-un editor de texte, putei realiza operaii simple, de genul nlocuirii irurilor de caractere sau mutrii i copierii acestora, ns orice lucru mai complex este greu de realizat.

Pentru a edita cum trebuie un fiier EPS, l putei converti ntr-un format editabil cu ajutorul programului pstoedit. Spre exemplu, pentru a obine un fiier editabil din formatul Xfig, dai urmtoarea comand:

$ pstoedit -f fig input.eps output.fig

Iar pentru a obine un fiier SVG pentru Inkscape, scriei n terminal:

$ pstoedit -f plot-svg input.eps output.svg

Cteodat programul pstoedit nu reuete s creeze formatul int (spre exemplu, cnd fiierul EPS conine informaii ce pot fi decupate). O cale mai robust de editare a fiierelor EPS const n convertirea mai nti a fiierului n PDF, urmat de importarea PDF-ului rezultat n Inkscape. Inkscape folosete biblioteca numit Cairo, care face o transformare de calitate ridicat a figurii EPS iniiale:

$ epstopdf input.eps
$ inkscape input.pdf

Cnd toate cele de mai sus eueaz, putei simplifica fiierul EPS nainte de a ncerca alte conversii, cu ajutorul programului eps2eps (vezi i seciunea urmtoare):

$ eps2eps input.eps input-e2.eps

Asta va converti toate fonturile n formate dinainte de generarea imaginilor, lucru care este de dorit cnd propunei manuscrise pentru publicare. Totui, ca dezavantaj, fonturile NU sunt convertite n linii, ci n hri de bii, ceea ce reduce calitatea fonturilor.

Convertirea EPS color n tonuri de gri[modificare]

Cteodat figurile EPS color trebuie convertite n figuri alb-negru sau cu tonuri de gri (grayscale) pentru a corespunde cerinelor de publicare. Asta se poate face cu programele eps2eps din pachetul Ghostscript i pscol:

$ eps2eps input.eps input-e2.eps
$ pscol -0gray input-e2.eps input-gray.eps
